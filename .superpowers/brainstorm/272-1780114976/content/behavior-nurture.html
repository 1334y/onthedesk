行为系统架构

三层决策架构
  ┌─────────────────────────────────────────────────┐
  │  Layer 1: 需求评估层（每 500ms 评估一次）         │
  │  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐        │
  │  │ 饥饿值  │ │ 体力值  │ │ 心情值  │ │ 好感度  │        │
  │  │ Hunger │ │ Energy │ │  Mood  │ │ Affection│       │
  │  └───┬───┘ └───┬───┘ └───┬───┘ └───┬───┘        │
  │      └─────────┴─────────┴─────────┘             │
  │                    ▼                             │
  │         需求优先级排序 → 触发对应行为              │
  └──────────────────────┬──────────────────────────┘
                         ▼
  ┌─────────────────────────────────────────────────┐
  │  Layer 2: 行为仲裁层（竞争式仲裁器）               │
  │                                                  │
  │   需求行为 > 用户交互 > 环境触发 > 随机探索        │
  │                                                  │
  │   例: 饥饿=90 → EAT 覆盖所有其他行为              │
  │       用户拖拽 → DRAG 覆盖除 EAT 外的一切          │
  └──────────────────────┬──────────────────────────┘
                         ▼
  ┌─────────────────────────────────────────────────┐
  │  Layer 3: 行为执行层（状态机 + 行为树混合）        │
  │                                                  │
  │   state → [sequence / selector / parallel]       │
  │        → AnimationController.play(anim, loop)    │
  │        → 行为后效果 (状态变化 / 粒子 / 气泡)       │
  └─────────────────────────────────────────────────┘

养成数值系统

四大状态维度
维度范围衰减影响
🍞 饥饿 Hunger 0-100 -1/10min 归零→强制觅食状态，拒绝玩耍
⚡ 体力 Energy 0-100 -1/5min(活动) 耗尽→强制睡眠，移动变慢
😊 心情 Mood 0-100 -1/15min 影响表情/动作丰富度
❤️ 好感 Affection 0-100 不衰减 累积型，解锁新动作/台词
好感度等级 & 解锁
等级好感解锁
🦀 陌生0-25基本动作(idle/walk/climb/sleep)
🦀 熟悉26-50坐窗口边晃腿、偶尔招手
🦀 亲近51-75跳舞、特殊表情、主动靠近鼠标
🦀 羁绊76-100彩蛋动作、稀有台词、传说皮肤
好感度获取：抚摸 +3、喂食 +5、聊天 +2、\\n长时间在线陪伴 +1/30min
